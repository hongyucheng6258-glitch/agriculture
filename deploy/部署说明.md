## 🖥️ 系统要求

### 服务器要求

- **操作系统**: Linux (CentOS 7+ / Ubuntu 18.04+)
- **CPU**: 2核以上
- **内存**: 4GB以上
- **硬盘**: 20GB以上可用空间

### 软件要求

- **Java**: JDK 11 或更高版本
- **Nginx**: 1.18+ (Web服务器)
- **Python 3**: 用于辅助脚本
- **SSH**: 用于远程连接

### 本地开发环境

- **Windows**: Windows 10+
- **Node.js**: 14+ (前端开发)
- **Maven**: 3.6+ (后端构建)
- **Git**: 可选，用于版本控制

## 🔧 手动部署

### 步骤 1: 构建前端

```bash
# 进入前端目录
cd frontend

# 安装依赖（首次运行）
npm install

# 构建生产版本
npm run build

# 构建完成后，dist 目录就是前端文件
```

### 步骤 2: 打包前端

```bash
# Windows PowerShell
Compress-Archive -Path dist\* -DestinationPath frontend_dist.zip

# Linux/Mac
cd dist && tar -czf ../frontend_dist.tar.gz .
```

### 步骤 3: 构建后端

```bash
# 进入后端目录
cd backend

# 使用Maven打包
mvn clean package -DskipTests

# 打包完成后，target目录下会有 jar 文件
# 文件名类似: aquaculture-system-1.0.0.jar
```

### 步骤 4: 上传到服务器

```bash
# 使用SCP上传
# 前端文件
scp frontend_dist.tar.gz root@8.147.71.153:/www/wwwroot/default/

# 后端文件
scp backend/target/aquaculture-system-1.0.0.jar root@8.147.71.153:/root/aquaculture/

# 配置文件
scp deploy/config/application.yml root@8.147.71.153:/root/aquaculture/
scp deploy/config/nginx.conf root@8.147.71.153:/etc/nginx/conf.d/
```

### 步骤 5: 在服务器上部署

#### 5.1 连接服务器

```bash
ssh root@8.147.71.153
```

#### 5.2 部署前端

```bash
# 进入前端目录
cd /www/wwwroot/default/

# 备份旧文件
tar -czf backup_$(date +%Y%m%d_%H%M%S).tar.gz index.html css js img fonts 2>/dev/null || true

# 清理旧文件
rm -rf index.html css js img fonts

# 解压新文件
tar -xzf frontend_dist.tar.gz

# 清理压缩包
rm frontend_dist.tar.gz
```

#### 5.3 部署后端

```bash
# 进入后端目录
cd /root/aquaculture/

# 停止旧服务
pkill -f "aquaculture-system-1.0.0.jar" 2>/dev/null || true

# 备份旧文件
cp aquaculture-system-1.0.0.jar aquaculture-system-1.0.0.jar.backup 2>/dev/null || true

# 确保目录存在
mkdir -p data data/export

# 启动新服务
nohup java -jar aquaculture-system-1.0.0.jar > app.log 2>&1 &

# 查看日志（可选）
tail -f app.log
```

#### 5.4 重启Nginx

```bash
# 检查配置
nginx -t

# 重启服务
nginx -s reload

# 或者
systemctl restart nginx
```

## 📝 常用命令

### 后端相关

```bash
# 启动服务
cd /root/aquaculture/
nohup java -jar aquaculture-system-1.0.0.jar > app.log 2>&1 &
```

```bash
# 查看文件列表
ls -la

tar -czf backup_$(date +%Y%m%d_%H%M%S).tar.gz index.html css js img fonts
```

```bash
# 检查配置
nginx -t

# 重启
nginx -s reload

# 启动/停止
systemctl start nginx
systemctl stop nginx
systemctl restart nginx
```

```bash
# 进入数据目录
cd /root/aquaculture/data/

# 备份数据库
cp aquaculture.db aquaculture.db.backup_$(date +%Y%m%d_%H%M%S)

sqlite3 aquaculture.db

# 查看表
sqlite3 aquaculture.db ".tables"

sqlite3 aquaculture.db "SELECT * FROM feed_stock;"
```

## 🔐 安全建议

1. **修改默认密码**: 首次登录后立即修改admin密码
2. **配置防火墙**: 只开放必要端口 (80, 22)
3. **定期备份**: 建议每日备份数据库
4. **日志轮转**: 配置日志轮转，避免磁盘占满
5. **使用HTTPS**: 生产环境建议配置SSL证书

## 🆘 故障排查

### 问题1: 前端页面无法访问

**检查:**
- Nginx是否运行: `systemctl status nginx`
- 防火墙是否开放80端口
- 前端文件是否存在: `ls -la /www/wwwroot/default/`

### 问题2: 后端API无法访问

**检查:**
- Java进程是否运行: `ps aux | grep java`
- 8080端口是否监听: `netstat -tuln | grep 8080`
- 查看后端日志: `tail -f /root/aquaculture/app.log`

### 问题3: 数据库问题

**检查:**
- 数据目录是否存在: `/root/aquaculture/data/`
- 数据库文件权限: `ls -la /root/aquaculture/data/`
- 数据库是否可写
